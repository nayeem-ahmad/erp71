# Implementation Stories

BMad implementation stories for the ERP71 platform, one file per story named
`<epic>-<story>-<kebab-title>.md` (e.g. `80-1-customer-crud.md`). Each story
follows the house format: `Story` (as-a/I-want/so-that), `Acceptance Criteria`,
`Data Model Changes`, `Tasks / Subtasks`, `Dev Notes`, and a `Dev Agent Record`.

- **Epic master list:** [../prd/epic-list.md](../prd/epic-list.md)
- **Sprint tracker:** [sprint-status.yaml](sprint-status.yaml)
- **Design specs (deeper technical designs):** [../superpowers/specs/](../superpowers/specs/)

> Moved here from `_bmad-output/implementation-artifacts/` (which was gitignored)
> so stories are versioned alongside the rest of the docs. The BMad
> `implementation_artifacts` path (`_bmad/bmm/config.yaml`) now points here, so
> newly generated stories land in this folder too.
